The Problem

The friction was the problem.
Not the work.

Most productivity systems fail not because the tools are bad but because using them requires too many steps. For someone managing ADHD alongside a full time job, a consulting business, a PhD, and multiple personal disciplines, opening five separate apps to log progress is not a system. It is a chore that stops getting done.

The work was happening. The tracking was not. And without tracking, the data that powers a good morning briefing does not exist. The whole system falls apart at the point of least friction.

Five separate apps with no unified view meant starting from scratch every time you opened one.

Without consistent check-ins the morning briefing had no real data to pull from.

Each app was built for a specific purpose but none of them talked to each other.

The system was smart but the friction of using it was defeating the purpose.

The Approach

Five apps. One system.
Zero manual coordination.

Instead of building one more app to replace the five, we designed a suite where each tool owned its specific domain and a shared intelligence layer connected all of them. The goal was to make check-ins feel like a natural part of using the app, not a separate administrative task.

01
Saunders HQ
Command center. Daily overview, priorities, and the single place to start every morning without opening anything else first.
02
Saunders Forge
Skill development tracker. Weekly check-ins for soldering, languages, and technical disciplines with progress visualization.
03
Saunders Scholar
PhD research companion. Module tracking, reading logs, and writing progress for a 13-module doctoral program.
04
Trigger Time
Firearms training tracker. Session logs, drill performance, and range data with historical progress tracking.
05
Saunders Focus
ADHD-friendly task management. Time-boxed work sessions, priority queuing, and distraction logging.
The Automation Layer

Every app writes to a shared Google Sheet when a check-in is completed. Make.com reads that sheet every morning at 9:45am, passes the data to the Claude API, and Claude generates a personalized three-sentence briefing based on actual activity. No manual input. No stale data. The briefing arrives via ntfy push notification before the day starts.

The Result

A system that runs itself
so you do not have to.

The best productivity system is one you forget is running.

The suite solved the original problem by removing the coordination burden entirely. Check-ins happen inside the tool you are already using. The briefing arrives without asking for it. The data stays current because the system updates itself.

What changed
Five disconnected apps became one coordinated operating system. Daily briefings went from generic to data-driven. Check-in rates improved because friction was removed not willpower demanded.
What it proved
AI-powered workflow automation does not have to be enterprise software. With the right architecture it can be personal, precise, and built by one person in a few weeks.
